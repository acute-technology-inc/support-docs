# MDIO

## What is MDIO?

MDIO (Management Data Input/Output) is a two-wire serial management interface defined in IEEE 802.3 for communicating with Ethernet physical layer (PHY) devices. Developed as part of the IEEE 802.3 standard, MDIO provides a standardized method for media access controllers (MACs) and other management entities to read status information, configure parameters, and control Ethernet PHY chips. The interface enables access to PHY registers containing link status, speed and duplex settings, auto-negotiation capabilities, error counters, and diagnostic information without interfering with the high-speed data path carrying Ethernet frames.

The MDIO specification exists in two main versions: Clause 22, the original specification supporting basic Ethernet speeds up to 1 Gbps with access to 32 registers per PHY device, and Clause 45, an enhanced specification introduced for 10 Gigabit Ethernet and beyond that provides indirect addressing for up to 65,536 registers per device. The interface uses only two signals—MDC (Management Data Clock) and MDIO (the bidirectional data line)—making it simple to implement while providing sufficient bandwidth for management operations. The clock rate typically runs at 2.5 MHz or lower, far slower than Ethernet data rates, as management operations are not time-critical and occur infrequently compared to data transmission.

MDIO is ubiquitous in Ethernet networking equipment ranging from consumer routers and switches to enterprise networking gear and embedded systems with Ethernet connectivity. The interface allows software drivers to initialize PHY devices during system startup, negotiate link parameters with remote devices, monitor link status for network diagnostics, and adjust PHY settings dynamically based on operating conditions. Modern Ethernet controllers integrate MDIO controllers that handle the low-level protocol details, presenting a simple register read/write interface to system software for managing one or more PHY devices on an MDIO bus.

## Technical Specifications

### Physical Layer

**Signal Lines:**
- **MDC (Management Data Clock)**: Clock signal generated by management entity (typically MAC)
- **MDIO**: Bidirectional data line, open-drain or tri-state with pull-up resistor

**Electrical Characteristics:**
- **Pull-up resistor**: 1.5 kΩ to 10 kΩ to VDD
- **Voltage levels (Clause 22)**: 5V CMOS or TTL compatible
- **Voltage levels (Clause 45)**: 1.2V, 1.8V, 2.5V, or 3.3V low-voltage CMOS
- **Clock frequency**: Up to 2.5 MHz maximum (400 ns minimum period)
- **Typical clock**: 1-2.5 MHz

**Bus Configuration:**
- **Topology**: Multi-drop bus supporting multiple PHY devices
- **Addressing**: Up to 32 PHY addresses (0-31)
- **Management entity**: Typically one MAC acting as master
- **PHY devices**: Multiple PHYs can share the same MDIO bus

### Clause 22 Protocol (IEEE 802.3 Clause 22)

**Frame Structure (32 bits):**

1. **Preamble**: 32 bits of '1' (MAC drives MDIO high while toggling MDC)
2. **Start of Frame (ST)**: 2 bits = '01'
3. **Operation Code (OP)**: 2 bits
   - '10' = Read
   - '01' = Write
4. **PHY Address (PHYADR)**: 5 bits (0-31)
5. **Register Address (REGADR)**: 5 bits (0-31)
6. **Turnaround (TA)**: 2 bits
   - Read: PHY drives MDIO after first bit
   - Write: MAC drives '10'
7. **Data**: 16 bits (register content)
8. **Idle**: MDIO released (pulled high)

**Total Frame**: 64 bits (including 32-bit preamble)

**Register Space:**
- 32 registers per PHY device (5-bit address)
- 32 possible PHY addresses (5-bit address)
- Total: 1024 addressable registers

**Clock Rate:**
- Maximum: 2.5 MHz
- Minimum period: 400 ns
- Typical: 1-2 MHz

### Clause 45 Protocol (IEEE 802.3 Clause 45)

**Enhanced Addressing:**
- Indirect addressing scheme using two-transaction access
- **MDIO Manageable Devices (MMDs)**: Up to 32 devices per port
- **Ports**: Up to 32 ports per MDIO bus
- **Registers**: 65,536 registers per MMD (16-bit address space)

**Frame Structure:**

**Address Frame (set address):**
1. **Preamble**: 32 bits of '1'
2. **Start of Frame (ST)**: 2 bits = '00'
3. **Operation Code (OP)**: 2 bits = '00' (Address)
4. **Port Address (PRTADR)**: 5 bits
5. **Device Type (DEVTYPE)**: 5 bits (MMD device type)
6. **Turnaround (TA)**: 2 bits
7. **Address**: 16 bits (register address within MMD)
8. **Idle**

**Read/Write Frame (perform operation):**
1. **Preamble**: 32 bits of '1'
2. **Start of Frame (ST)**: 2 bits = '00'
3. **Operation Code (OP)**: 2 bits
   - '11' = Read
   - '01' = Write
   - '10' = Read-Increment (post-increment address)
4. **Port Address (PRTADR)**: 5 bits
5. **Device Type (DEVTYPE)**: 5 bits
6. **Turnaround (TA)**: 2 bits
7. **Data**: 16 bits
8. **Idle**

**Two-Transaction Access:**
1. Address frame sets target register address
2. Read/Write frame accesses the addressed register
3. Optional Read-Increment for burst access

### Common PHY Registers (Clause 22)

**Standard Registers (defined by IEEE 802.3):**
- **Register 0**: Control (reset, loopback, speed select, auto-negotiation enable)
- **Register 1**: Status (link status, auto-negotiation ability, remote fault)
- **Register 2**: PHY Identifier (high 16 bits)
- **Register 3**: PHY Identifier (low 16 bits)
- **Register 4**: Auto-Negotiation Advertisement
- **Register 5**: Auto-Negotiation Link Partner Ability
- **Register 6**: Auto-Negotiation Expansion
- **Register 7**: Next Page Transmit
- **Registers 16-31**: Vendor-specific extended registers

## Common Applications

MDIO is used in all Ethernet networking systems for PHY management:

- **Network routers**: Home, enterprise, and carrier-grade routers
- **Ethernet switches**: Unmanaged, managed, and layer-3 switches
- **Embedded systems**: IoT devices, industrial controllers with Ethernet
- **Single-board computers**: Raspberry Pi, BeagleBone with Ethernet
- **Automotive Ethernet**: In-vehicle networking (BroadR-Reach, 100BASE-T1)
- **Network interface cards**: Desktop and server NICs
- **Media converters**: Fiber-to-copper and speed converters
- **PoE devices**: Power over Ethernet switches and injectors
- **Industrial Ethernet**: EtherCAT, PROFINET, EtherNet/IP equipment
- **Telecommunications**: Carrier Ethernet equipment, DSLAMs
- **Data center equipment**: Top-of-rack switches, network appliances
- **Network test equipment**: Protocol analyzers, traffic generators
- **IP cameras**: Surveillance cameras with Ethernet connectivity
- **VoIP phones**: Network-connected telephone systems
- **Smart home devices**: Network-connected appliances and controls

## Decoder Configuration

When configuring a logic analyzer to decode MDIO signals:

### Channel Assignment

**Required Signals:**
- **MDC**: Management Data Clock (required)
- **MDIO**: Bidirectional data line (required)

Both signals should be probed at the PHY device pins or along the MDIO bus traces.

### Protocol Parameters

- **Clause version**: Select Clause 22 or Clause 45
- **Clock frequency**: Set expected MDC frequency (typically 1-2.5 MHz)
- **PHY address**: Optionally filter by specific PHY address
- **Voltage level**: Set input threshold for logic levels (1.2V, 1.8V, 3.3V, or 5V)

### Decoding Options

- **Frame type display**: Show Read, Write, Address operations
- **PHY/Port address**: Display target device address
- **Register address**: Show register number being accessed (Clause 22) or MMD device type (Clause 45)
- **Data value**: Display 16-bit register content in hex/binary
- **Register name**: Decode standard register numbers to names (Control, Status, PHY ID, etc.)
- **Bit field decoding**: Parse register bit fields (link status, speed, duplex, etc.)
- **Preamble display**: Optionally show/hide 32-bit preamble
- **Turnaround handling**: Indicate bus direction change

### Trigger Configuration

- **Frame start**: Trigger on start of MDIO frame (ST field)
- **Specific operation**: Trigger on Read or Write operation
- **PHY address**: Trigger when specific PHY address is accessed
- **Register address**: Trigger on access to specific register (e.g., Status register)
- **Data pattern**: Trigger when specific data value is written or read
- **Link status change**: Trigger on Status register read indicating link change

### Sampling Requirements

**Minimum Sampling Rate:**
- At least 10× the MDC frequency
- Example: 2.5 MHz MDC requires 25 MHz sampling minimum

**Recommended Sampling Rate:**
- 20× MDC frequency for clear waveform visualization
- Example: 2.5 MHz MDC requires 50 MHz sampling

**Buffer Depth:**
- Each frame is 64 bits (with preamble)
- Typical capture: 100-1000 frames = 6.4 kb to 64 kb minimum

### Analysis Tips

When analyzing MDIO communications:

1. **Identify Clause version**: Check ST field—'01' for Clause 22, '00' for Clause 45
2. **Monitor initialization**: Observe register writes during PHY initialization after reset
3. **Check PHY ID**: Read registers 2 and 3 to verify PHY device identification
4. **Link status polling**: Watch for periodic reads of Status register (register 1)
5. **Auto-negotiation**: Observe registers 4, 5, 6 during link negotiation
6. **Turnaround timing**: Verify proper tristate/drive timing during TA field
7. **Multi-PHY systems**: Identify different PHY addresses on shared MDIO bus
8. **Clause 45 indirect access**: For Clause 45, pair Address and Read/Write transactions

### Common Protocol Sequences

**PHY Initialization (Clause 22):**
1. Write Control register (0) to reset PHY (bit 15 = 1)
2. Wait for reset to complete
3. Read PHY ID registers (2 and 3)
4. Configure Control register (speed, duplex, auto-negotiation)
5. Write Auto-Negotiation Advertisement register (4)
6. Start auto-negotiation (write Control register, bit 12 = 1)

**Link Status Monitoring:**
1. Periodic read of Status register (1)
2. Check link status bit (bit 2)
3. If link changes, read additional registers for details

**Read Transaction (Clause 22):**
1. Preamble: 32 bits of '1'
2. ST = '01'
3. OP = '10' (Read)
4. PHYADR = PHY address (e.g., '00001')
5. REGADR = Register address (e.g., '00001' for Status)
6. TA = 'Z0' (PHY takes control after first bit)
7. DATA = 16-bit register value driven by PHY
8. MDIO released

**Write Transaction (Clause 22):**
1. Preamble: 32 bits of '1'
2. ST = '01'
3. OP = '01' (Write)
4. PHYADR = PHY address
5. REGADR = Register address (e.g., '00000' for Control)
6. TA = '10'
7. DATA = 16-bit value to write
8. MDIO released
