# SMI (Serial Management Interface)

## What is SMI?

### Overview

SMI (Serial Management Interface), also commonly known as MDIO (Management Data Input/Output), is a 2-wire serial management protocol defined in IEEE 802.3 Clause 22 and Clause 45 for configuring and monitoring Ethernet PHY (Physical Layer) transceivers. Developed as part of the MII (Media Independent Interface) specification, SMI provides a standardized method for a network controller or switch to access the internal registers of Ethernet PHYs, enabling functions such as link status monitoring, speed and duplex configuration, auto-negotiation control, cable diagnostics, and vendor-specific feature management. The protocol uses a simple master-slave architecture where the MAC or management entity acts as the master (also called Station Management Entity or STA) and the PHY acts as the slave.

The SMI bus consists of two signals: MDC (Management Data Clock), a clock signal generated by the master at up to 2.5 MHz, and MDIO (Management Data Input/Output), a bidirectional data line shared between master and PHY. Communication occurs through frame-based transactions where the master initiates read or write operations to specific PHY registers using a defined frame format. Clause 22 (original MII) supports 5-bit PHY addresses and 5-bit register addresses (32 registers per PHY), while the extended Clause 45 format supports 5-bit port/device addresses and 16-bit register addresses (65,536 registers per device), accommodating the more complex register sets found in Gigabit and 10-Gigabit Ethernet PHYs.

SMI/MDIO has become ubiquitous in Ethernet systems across all segments—from embedded 10/100 Mbps controllers to enterprise 10/100GbE switches. The protocol is supported by virtually all Ethernet PHY manufacturers and is commonly implemented as a shared bus where a single MDC/MDIO pair connects a MAC to multiple PHYs, with each PHY identified by a unique 5-bit physical address (0-31). Understanding SMI is essential for debugging Ethernet connectivity issues, validating PHY configuration, developing network drivers, and troubleshooting link negotiation problems.

### Key Features

- **2-Wire Interface**: MDC (clock) and MDIO (bidirectional data)
- **PHY Configuration**: Read/write access to PHY registers
- **Standard Protocol**: IEEE 802.3 Clause 22 and Clause 45
- **Multi-PHY Support**: Shared bus with up to 32 PHYs (5-bit address)
- **Low Speed**: Up to 2.5 MHz MDC clock
- **Simple Protocol**: Frame-based transactions with clear structure
- **Register Access**: Configure speed, duplex, auto-negotiation, cable diagnostics
- **Link Status**: Monitor link state, errors, statistics
- **Vendor Extensions**: Access manufacturer-specific features
- **Backward Compatible**: Clause 45 supports Clause 22 frames

## Technical Specifications

### Signal Description

**SMI/MDIO Signals** (2 wires)

- **MDC (Management Data Clock)**: Clock output from master (STA)
- **MDIO (Management Data Input/Output)**: Bidirectional data line

**Electrical Characteristics**
- **Voltage Levels**: Typically 2.5V or 3.3V CMOS
- **MDC Frequency**: DC to 2.5 MHz maximum (often 1-2.5 MHz)
- **MDIO Drive**: Open-drain or push-pull (open-drain requires pull-up)
- **Pull-up**: 1.5kΩ to 10kΩ resistor on MDIO (if open-drain)

### Clause 22 Frame Format (Original MII)

**Frame Structure** (32 or 64 bits total)

```
| PRE | ST | OP | PHYAD | REGAD | TA | DATA      | IDLE |
| 32  | 2  | 2  | 5     | 5     | 2  | 16        | ...  |
```

**Field Definitions**

- **PRE (Preamble)**: 32 bits of '1' (optional, for synchronization)
- **ST (Start of Frame)**: 2 bits = '01' (Clause 22 indicator)
- **OP (Operation Code)**: 2 bits
  - '10' = Read
  - '01' = Write
- **PHYAD (PHY Address)**: 5 bits (0-31, identifies target PHY)
- **REGAD (Register Address)**: 5 bits (0-31, specifies register)
- **TA (Turnaround)**: 2 bits
  - Read: Master drives 'ZZ' (tri-state), PHY drives '10'
  - Write: Master drives '10'
- **DATA**: 16 bits (register data, LSB first)
- **IDLE**: MDIO returns to high (tri-state with pull-up)

**Read Transaction Timing**
```
MDC:  __|‾|__|‾|__|‾|__|‾|__|‾|__|‾|__|‾|__|‾|__|‾|__
MDIO: |PRE....|01|10|PHYAD |REGAD |ZZ|  DATA (from PHY) |Z
                              ↑ Master releases, PHY drives
```

**Write Transaction Timing**
```
MDC:  __|‾|__|‾|__|‾|__|‾|__|‾|__|‾|__|‾|__|‾|__|‾|__
MDIO: |PRE....|01|01|PHYAD |REGAD |10|  DATA (from Master) |Z
                                          ↑ Master drives data
```

### Clause 45 Frame Format (Extended)

**Clause 45 adds indirect addressing for larger register spaces**

**Address Frame** (to set register address)
```
| PRE | ST | OP | PRTAD | DEVAD | TA | ADDRESS   | IDLE |
| 32  | 2  | 2  | 5     | 5     | 2  | 16        | ...  |
```

**Data Frame** (to read/write data)
```
| PRE | ST | OP | PRTAD | DEVAD | TA | DATA      | IDLE |
| 32  | 2  | 2  | 5     | 5     | 2  | 16        | ...  |
```

**Clause 45 Operation Codes**
- **ST**: '00' (Clause 45 indicator)
- **OP**:
  - '00' = Address (set register address)
  - '01' = Write (write to previously set address)
  - '10' = Read (post-increment address)
  - '11' = Read (current address, no increment)

**PRTAD**: Port/PHY Address (5 bits)  
**DEVAD**: Device/MMD (MDIO Manageable Device) Address (5 bits)

**Clause 45 Two-Step Access**
1. Send Address frame with OP='00' to set 16-bit register address
2. Send Write (OP='01') or Read (OP='11') frame to access data at that address

### Timing Specifications

**MDC Clock**
- **Maximum Frequency**: 2.5 MHz (400 ns period minimum)
- **Typical Frequency**: 1 MHz to 2.5 MHz
- **Duty Cycle**: 40-60% acceptable
- **Minimum Low/High Time**: 160 ns each

**MDIO Timing** (relative to MDC)
- **Setup Time**: 10 ns minimum (MDIO stable before MDC rising)
- **Hold Time**: 10 ns minimum (MDIO stable after MDC rising)
- **Output Delay**: Maximum 300 ns (PHY driving MDIO after MDC)

**Transaction Timing**
- **Clause 22 Read**: 32 (preamble) + 32 (frame) = 64 MDC clocks
- **Clause 22 Write**: 32 (preamble) + 32 (frame) = 64 MDC clocks
- **Preamble Optional**: Some PHYs work without preamble (32 clocks shorter)

### Common PHY Registers (Clause 22)

**Basic Registers** (IEEE 802.3 standard)

- **Register 0 (Control)**:
  - Bit 15: Reset
  - Bit 14: Loopback
  - Bit 13: Speed selection (MSB)
  - Bit 12: Auto-negotiation enable
  - Bit 11: Power down
  - Bit 9: Restart auto-negotiation
  - Bit 8: Duplex mode (1 = full, 0 = half)
  - Bit 6: Speed selection (LSB)

- **Register 1 (Status)**:
  - Bit 5: Auto-negotiation complete
  - Bit 4: Remote fault
  - Bit 3: Auto-negotiation ability
  - Bit 2: Link status (1 = up)
  - Bit 1: Jabber detect
  - Bit 0: Extended capability

- **Register 2 & 3**: PHY Identifier (manufacturer OUI)
- **Register 4**: Auto-Negotiation Advertisement
- **Register 5**: Auto-Negotiation Link Partner Ability
- **Register 6**: Auto-Negotiation Expansion

**Extended/Vendor Registers** (16-31): Vendor-specific features

## Decoder Configuration

When analyzing SMI/MDIO traffic with a logic analyzer, configure the following parameters:

### Signal Connections

**Minimum Configuration** (2 channels)
- MDC - Management Data Clock
- MDIO - Management Data I/O

### Sampling Requirements

- **Minimum Sample Rate**: 20 MS/s (8× the 2.5 MHz maximum MDC)
- **Recommended**: 50-100 MS/s for reliable capture and timing analysis

### Decoder Parameters

- **Protocol Version**: Clause 22 or Clause 45
- **Preamble**: Enabled (32 bits) or Disabled (for faster transactions)
- **Clock Edge**: Rising edge (standard for MDIO capture)
- **PHY Address**: Specify if filtering for specific PHY (0-31)
- **Register Decoding**: Enable interpretation of standard registers

### Display Options

- Show transaction type (Read/Write)
- Display PHY address being accessed
- Show register address
- Display data value (hex or binary)
- Decode standard register bit fields (Control, Status, etc.)
- Annotate preamble, start bits, operation codes
- Indicate turnaround phase and bus direction
- Show Clause 45 address+data transaction pairs

### Trigger Settings

- Trigger on start bits (ST='01' for C22, ST='00' for C45)
- Trigger on specific PHY address
- Trigger on specific register address access
- Trigger on operation type (Read or Write)
- Trigger on specific data patterns

### Analysis Tips

1. **Identify Protocol Version**
   - ST='01' indicates Clause 22
   - ST='00' indicates Clause 45
   - Check after preamble

2. **Verify PHY Address**
   - Extract 5-bit PHYAD field
   - Ensure matches expected PHY physical address
   - Common values: 0, 1, 2 (often hardware-strapped)

3. **Decode Common Registers**
   - Register 0: Control (reset, auto-neg enable, speed, duplex)
   - Register 1: Status (link up/down, auto-neg complete)
   - Register 2/3: PHY ID (manufacturer identification)
   - Register 4/5: Auto-negotiation advertisement and partner ability

4. **Monitor Auto-Negotiation**
   - Watch for writes to Register 0 (enable/restart auto-neg)
   - Monitor Register 1 for auto-neg complete
   - Check Register 5 for link partner capabilities
   - Verify negotiated speed/duplex matches expected

5. **Check Link Status**
   - Read Register 1 repeatedly (polling by MAC)
   - Bit 2 indicates link status
   - Reads often occur every 1-2 seconds

6. **Timing Analysis**
   - Verify MDC frequency ≤ 2.5 MHz
   - Check MDIO setup/hold times relative to MDC
   - Measure transaction duration

7. **Turnaround Phase**
   - Read: Master tri-states, PHY drives '10' then data
   - Write: Master drives '10' continuously
   - Look for bus contention (both driving)

### Common Issues

**No Communication / Bus Idle**
- **Symptom**: No MDC activity or constant MDIO high
- **Cause**: MAC not initializing PHY, driver issue, hardware fault
- **Solution**: Check MAC driver initialization, verify PHY powered

**Invalid Preamble**
- **Symptom**: Not 32 '1' bits
- **Cause**: Preamble suppression enabled (valid), or corruption
- **Solution**: Check if preamble suppression intentional

**Wrong PHY Address**
- **Symptom**: No response from PHY (MDIO stays high during TA)
- **Cause**: PHY address mismatch, incorrect strapping pins
- **Solution**: Verify PHY address strapping, check schematic

**No Turnaround Response**
- **Symptom**: PHY doesn't drive MDIO during read TA
- **Cause**: PHY not responding, wrong address, PHY in reset
- **Solution**: Verify PHY address, check PHY power and reset

**Data Corruption**
- **Symptom**: Read data doesn't match expected values
- **Cause**: Timing violations, MDIO line noise, pull-up issues
- **Solution**: Check signal integrity, verify pull-up resistor, reduce MDC speed

**Link Not Establishing**
- **Symptom**: Register 1 shows link down
- **Cause**: PHY misconfiguration, cable issue, auto-neg failure
- **Solution**: Verify Register 0 settings, check cable, analyze auto-neg

**Bus Contention**
- **Symptom**: MDIO conflicts during turnaround
- **Cause**: Master not tri-stating properly, PHY driving too early
- **Solution**: Check MDC/MDIO timing, verify open-drain vs. push-pull

### Advanced Analysis

**PHY Initialization Sequence**
- Capture full power-up sequence
- Watch for reset (Register 0, bit 15)
- Monitor auto-negotiation setup
- Track link establishment time

**Auto-Negotiation Analysis**
- Decode advertisement registers (4 and 5)
- Verify compatible capabilities
- Check for forced speed/duplex override
- Monitor negotiation restart events

**Cable Diagnostics**
- Some PHYs support TDR (Time Domain Reflectometry) via registers
- Analyze vendor-specific registers for cable length, faults
- Monitor extended register accesses

**Performance Monitoring**
- Track register access frequency
- Identify polling intervals
- Measure transaction overhead
- Optimize SMI bus utilization

**Multi-PHY Systems**
- Identify which PHY addresses are in use
- Monitor address conflicts
- Verify each PHY responds correctly
- Analyze shared bus arbitration
