# LCD1602 (HD44780-Compatible Character LCD)

## What is LCD1602?

### Overview

LCD1602 refers to a ubiquitous 16-character by 2-line alphanumeric LCD display module based on the Hitachi HD44780 LCD controller or compatible derivatives (Samsung KS0066, SPLC780, ST7066, etc.). Introduced in the 1980s, the HD44780 controller and its interface protocol have become the de facto standard for character-based LCD displays in embedded systems, hobbyist projects, industrial equipment, and consumer electronics. These displays are extremely popular due to their low cost ($2-5), simple parallel or I2C interface, built-in character generator ROM (supports ASCII and custom characters), and extensive software library support across virtually every microcontroller platform.

The LCD1602 interface typically operates in either 4-bit or 8-bit parallel mode, using control signals RS (Register Select), R/W (Read/Write), and E (Enable) along with the data bus (D0-D7 for 8-bit, D4-D7 for 4-bit). The controller includes an instruction register for commands (clear display, cursor control, display shift) and a data register for ASCII characters to be displayed. The module contains onboard character generator ROM (CGROM) with 208 pre-defined characters including uppercase/lowercase letters, numbers, symbols, and Japanese katakana characters, plus CGRAM (Character Generator RAM) for 8 user-definable custom characters. The controller handles all the low-level LCD driving, multiplexing, and refresh automatically.

LCD1602 modules are found in countless applications: Arduino and Raspberry Pi projects, industrial controllers, test equipment front panels, home automation displays, HVAC thermostats, vending machines, and educational electronics kits. The interface is so well-established that thousands of code libraries exist for every microcontroller (Arduino LiquidCrystal library, CircuitPython, MicroPython, etc.), and I2C backpack adapters allow control with just 2 wires instead of 6-11. While graphical TFT and OLED displays have become more common, LCD1602 remains popular for simple text-based interfaces due to its low power consumption, readability in bright light, and decades of proven reliability.

### Key Features

- **16×2 Character Display**: 16 characters per line, 2 lines
- **HD44780 Controller**: Industry-standard controller
- **Built-in Character Set**: 208 characters in CGROM (ASCII + symbols)
- **Custom Characters**: 8 user-definable characters in CGRAM
- **4-bit or 8-bit Mode**: Flexible parallel interface
- **I2C Compatible**: Via PCF8574/PCF8574A I2C expander backpack
- **Low Power**: ~1-3 mA typical (backlight additional 15-120 mA)
- **Adjustable Contrast**: Potentiometer or PWM control
- **Backlight Options**: LED backlight (white, blue, green, RGB)
- **Simple Protocol**: Easy to interface with any microcontroller

## Technical Specifications

### Signal Description

**Parallel Interface Signals** (8-bit mode)

- **RS (Register Select)**: 0 = Command/instruction, 1 = Data/character
- **R/W (Read/Write)**: 0 = Write to LCD, 1 = Read from LCD
- **E (Enable)**: Data latch on falling edge (active high)
- **D0-D7**: 8-bit parallel data bus

**4-bit Mode Signals** (to save pins)

- **RS**: Register select
- **R/W**: Read/write (often tied to GND for write-only)
- **E**: Enable
- **D4-D7**: 4-bit data bus (upper nibble only)

**Power and Auxiliary**

- **VDD**: Power supply +5V (or +3.3V for some modules)
- **VSS**: Ground (0V)
- **V0 (VEE)**: Contrast adjustment (voltage divider from pot)
- **A (Anode)**: Backlight LED anode (+)
- **K (Cathode)**: Backlight LED cathode (-)

**Typical Pin Configuration** (16-pin header)

| Pin | Symbol | Function |
|-----|--------|----------|
| 1 | VSS | Ground |
| 2 | VDD | Power +5V |
| 3 | V0 | Contrast adjust |
| 4 | RS | Register select |
| 5 | R/W | Read/Write |
| 6 | E | Enable |
| 7-10 | D0-D3 | Data bits 0-3 (not used in 4-bit mode) |
| 11-14 | D4-D7 | Data bits 4-7 |
| 15 | A | Backlight anode |
| 16 | K | Backlight cathode |

### Timing Specifications

**Enable (E) Pulse Timing** (write operation)

- **Enable High Time (PWeh)**: ≥450 ns (enable pulse width)
- **Enable Low Time (PWel)**: ≥500 ns (between pulses)
- **Enable Cycle Time (tCYC)**: ≥1000 ns (1 µs minimum)
- **Address Setup Time (tAS)**: ≥60 ns (RS, R/W stable before E high)
- **Address Hold Time (tAH)**: ≥10 ns (RS, R/W held after E low)
- **Data Setup Time (tDSW)**: ≥195 ns (data stable before E falling edge)
- **Data Hold Time (tH)**: ≥10 ns (data held after E falling)

**Command Execution Times**

- **Clear Display (0x01)**: ~1.52 ms
- **Return Home (0x02)**: ~1.52 ms
- **Write Character**: ~37 µs typical
- **Read Busy Flag**: ~0.1 µs
- **Power-On to Ready**: ~15 ms (wait after VDD stable)

### Command Set

**Display Control Commands**

- **0x01**: Clear display (fill with spaces, cursor home)
- **0x02**: Return home (cursor to position 0, no clear)
- **0x04-0x07**: Entry mode set (cursor direction, display shift)
- **0x08-0x0F**: Display ON/OFF, cursor control, blink control
- **0x10-0x1F**: Cursor/display shift (move cursor or scroll)
- **0x20-0x3F**: Function set (4/8-bit, 1/2 lines, 5×8/5×10 font)
- **0x40-0x7F**: Set CGRAM address (define custom characters)
- **0x80-0xFF**: Set DDRAM address (cursor position for text)

**Common Initialization Sequence** (4-bit mode)

```
1. Wait >15 ms after VDD rises to 4.5V
2. Send 0x03 (function set 8-bit): wait 4.1 ms
3. Send 0x03 again - wait 100 µs
4. Send 0x03 again
5. Send 0x02 (function set 4-bit mode)
6. Send 0x28 (4-bit, 2 lines, 5×8 font)
7. Send 0x0C (display on, cursor off)
8. Send 0x06 (entry mode: increment cursor, no shift)
9. Send 0x01 (clear display)
10. Ready for use
```

### 4-Bit Mode Operation

**Sending a Byte in 4-Bit Mode**

1. Set RS (0 for command, 1 for data)
2. Send upper nibble (bits 7-4) on D7-D4
3. Pulse E high then low to latch upper nibble
4. Send lower nibble (bits 3-0) on D7-D4
5. Pulse E high then low to latch lower nibble

**Example: Write 'A' (0x41)**

```
RS=1, D7-D4=0x4 (upper), E pulse
RS=1, D7-D4=0x1 (lower), E pulse
```

### Display Addressing

**DDRAM (Display Data RAM) Addresses**

- **Line 1**: Addresses 0x00-0x0F (16 characters)
- **Line 2**: Addresses 0x40-0x4F (16 characters)

To set cursor to line 2, character 5:  
Send command: 0x80 | (0x40 + 4) = 0xC4

**Custom Characters (CGRAM)**

- 8 custom characters available (0x00-0x07)
- Each character is 5×8 pixels (5 columns, 8 rows)
- Define pattern: write 8 bytes to CGRAM addresses 0x40-0x7F
- Display custom character: write 0x00-0x07 to DDRAM

## Decoder Configuration

When analyzing LCD1602 communication with a logic analyzer:

### Signal Connections

**4-Bit Mode** (most common, 6 channels)
- RS - Register select
- R/W - Read/write (if not tied to GND)
- E - Enable
- D4, D5, D6, D7 - Data lines

**8-Bit Mode** (11 channels)
- Add D0, D1, D2, D3

### Sampling Requirements

- **Minimum Sample Rate**: 5 MS/s
- **Recommended**: 10-20 MS/s for timing analysis

### Decoder Parameters

- **Mode**: 4-bit or 8-bit
- **RS Polarity**: 0 = Command, 1 = Data
- **E Polarity**: Active high (latch on falling edge)

### Display Options

- Decode commands (e.g., 0x01 = Clear Display)
- Show ASCII characters being written
- Display DDRAM addresses (cursor positions)
- Annotate timing (execution delays)

### Trigger Settings

- Trigger on E falling edge
- Trigger on RS transition (command to data)
- Trigger on specific command (e.g., 0x01 clear)

### Analysis Tips

1. **Verify Initialization**: Check for proper initialization sequence at startup
2. **Check Timing**: Ensure E pulse width ≥450 ns and cycle time ≥1 µs
3. **Decode Commands**: Identify display control, cursor positioning, clear display
4. **Read Characters**: Decode ASCII data bytes (RS=1)
5. **Monitor Delays**: Verify delays after slow commands (clear, home)

### Common Issues

- **Garbled Display**: Wrong nibble order in 4-bit mode, timing too fast
- **No Display**: Missing initialization, wrong contrast voltage
- **Wrong Characters**: Incorrect DDRAM addressing, line 2 not at 0x40
- **Timing Violations**: E pulse too short, data not stable
